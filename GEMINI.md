# Project Instructions: Dongttok (동똑이)

Welcome to the Dongttok project! This repository contains the source code for an AI-powered chatbot assistant designed for Dongguk University students. It provides reliable information about campus life, academic rules, notices, and staff contacts using RAG (Retrieval-Augmented Generation) technology.

## Project Overview

Dongttok is a multi-service application consisting of three main components:
1.  **Frontend:** A React (TypeScript) application built with Vite.
2.  **Backend (Proxy/Auth):** An ASP.NET Core 9.0 server handling user authentication, chat sessions, and proxying requests to the AI engine.
3.  **RAG Engine:** A Python (FastAPI) service that implements the retrieval and generation logic using LangChain, ChromaDB, and OpenAI GPT-4o-mini.

### Key Technologies
- **Frontend:** React 19, Vite, TypeScript, Bootstrap, React Markdown.
- **Backend:** .NET 9, Entity Framework Core (PostgreSQL), JWT Authentication.
- **AI/RAG:** Python 3.10+, FastAPI, LangChain, ChromaDB, Redis (Caching), KURE-v1 (Embedding), TF-IDF.
- **Infrastructure:** Docker, Docker Compose, Nginx.

## Directory Structure
- `src/RenuxServer/`: Main C# backend and home of the React frontend.
- `src/RenuxServer/wwwroot/frontend/`: React frontend source.
- `src/RAG/`: Python RAG engine service.
- `src/RAG/api/`: FastAPI route definitions.
- `src/RAG/src/`: Core RAG logic, search algorithms, and database models.
- `src/RAG/data/`: Raw and processed campus data.
- `doc/`: Project documentation and architecture diagrams.

## Building and Running

### Prerequisites
- Docker and Docker Compose
- .NET 9 SDK (for local development)
- Python 3.10+ (for local development)
- Node.js (for local development)

### Running with Docker (Recommended)
The entire stack can be launched using Docker Compose from the `src/RenuxServer` directory.

1.  Navigate to the server directory:
    ```bash
    cd src/RenuxServer
    ```
2.  Create a `.env` file based on the instructions in `src/README.md`.
3.  Start the services:
    ```bash
    docker compose up --build
    ```
4.  Access the application at `http://localhost:8080`.

### Local Development
- **Frontend:**
  ```bash
  cd src/RenuxServer/wwwroot/frontend
  npm install
  npm run dev
  ```
- **RAG Engine:**
  ```bash
  cd src/RAG
  pip install -r requirements.txt
  python -m uvicorn api.rag_service:app --reload --port 8000
  ```
- **Backend:**
  ```bash
  cd src/RenuxServer
  dotnet run
  ```

## Development Conventions

### General
- **Security:** Never commit secrets. Use `.env` files and ensure they are ignored by git.
- **Documentation:** Keep `README.md` and diagrams in `doc/` updated with architectural changes.

### Frontend (React)
- Use TypeScript for all new components.
- Prefer functional components and hooks.
- Follow the existing component structure in `src/components/`.

### Backend (C#)
- Follow standard C# naming conventions (PascalCase for classes and methods).
- Use AutoMapper for DTO mapping.
- Keep business logic in services or API definitions where appropriate.

### AI Engine (Python)
- Maintain type hints where possible.
- Use `logging` for tracking RAG pipeline execution.
- When adding new data sources, update `src/RAG/src/pipelines/ingest.py` and ensure proper metadata tagging for filtering.

## Testing
- **Frontend:** Run `npm run lint` before committing.
- **RAG Engine:** Evaluation scripts are available in `src/RAG/scripts/evaluate_rag.py`.
- **General:** Manual verification of the chat flow and admin dashboard is required after major changes.
