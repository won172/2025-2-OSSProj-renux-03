## II. 프로젝트 수행 결과

### 1. 프로젝트 개요

#### 1.1 개발 동기 및 목적
- **동기:** 학교 홈페이지의 공지사항, 학칙, 학사일정, 교과과정 정보가 여러 곳에 분산되어 있어 학생들이 원하는 정보를 신속하게 찾기 어려움. 기존 동국대학교 챗봇 서비스는 템플릿 기반의 응답 구조로, 복합적인 맥락을 요구하는 질문이나 정보의 최신성 반영에 한계가 있음.
- **목적:** 자연어 질문을 통해 학교 생활에 필요한 정보를 쉽고 정확하게 제공하는 생성형 AI 기반의 챗봇 서비스를 개발하여 학생들의 편의를 증진하고 정보 접근성을 높임.

#### 1.2 필요성
- 기존 챗봇의 링크 기반 응답 방식은 사용자가 직접 문서를 다시 탐색해야 하는 번거로움이 있음.
- 사전에 정해진 답변이 아닌, 사용자의 질문 의도를 파악하고 맥락에 맞는 직접적인 답변을 제공하는 AI 시스템이 필요함.
- 학칙, 규정 등 복잡한 문서를 요약하고 핵심만 전달해줄 수 있는 도구가 요구됨.

#### 1.3 개발 목표
- **정확성:** RAG(검색 증강 생성) 기술을 도입하여 할루시네이션(거짓 답변)을 최소화하고 근거(출처)가 명확한 답변 제공.
- **최신성:** 1일 4회 이상의 주기적인 데이터 업데이트를 통해 최신 학사 정보 제공.
- **개인화:** 학과별, 개인별 맞춤형 답변 제공 기능 구현.

### 2. 최종결과물 소개

* **Renux RAG 챗봇 시스템 (동똑이)**
    * 사용자의 자연어 질문을 이해하고 관련 문서를 검색하여 답변하는 대화형 인터페이스.
    * **주요 기능:**
        * **자연어 질의응답:** 학사일정, 공지사항, 학칙, 교과과정 등 통합 검색 및 답변 생성.
        * **출처 제공:** 답변의 신뢰도를 높이기 위해 참조한 문서의 출처(URL, 문서명) 명시.
        * **멀티턴 대화:** 이전 대화의 맥락을 기억하여 연속적인 질문과 답변 가능.
    * **화면 구성:**
        * 메인 페이지: 직관적인 채팅 인터페이스 제공.
        * 회원가입/로그인: 학생 인증을 통한 맞춤형 서비스 제공 기반 마련.
        * 채팅 내역 관리: 과거 대화 내용을 저장하고 조회할 수 있는 기능.

### 3. 프로젝트 추진 내용

#### 3.1 프로젝트 진행과정
* **기획 단계:** 기존 동국대 챗봇 및 타 대학 챗봇 분석, 사용자 요구사항 정의, 기술 스택 선정.
* **설계 단계:** RAG 파이프라인 설계, DB 스키마(ERD) 설계, 시스템 아키텍처 구성.
* **개발 단계:**
    * **데이터 파이프라인:** 공지사항 크롤러 구현, 데이터 전처리 및 정제 로직 개발.
    * **RAG 엔진:** ChromaDB + TF-IDF 하이브리드 검색 엔진 구현, 한국어 특화 임베딩(KURE-v1) 적용.
    * **백엔드:** .NET 9.0 기반의 API 서버 구축, PostgreSQL 연동, Redis 세션 관리 구현.
    * **프론트엔드:** React 기반의 웹 클라이언트 개발, 채팅 UI/UX 구현.
* **테스트 및 최적화:** 청크 분할 전략 개선, LLM 프롬프트 엔지니어링, 로컬 및 클라우드 배포 테스트.

#### 3.2 프로젝트 구현과정

**1. 시스템 아키텍처**
* **Frontend:** React, TypeScript, Vite.
* **Backend:** .NET 9.0 (RenuxServer), FastAPI (AI Service).
* **Database:** PostgreSQL (User/Chat Data), ChromaDB (Vector Store), Redis (Session/Cache).
* **AI Model:** OpenAI GPT-4o mini (Generation), KURE-v1 (Embedding).

**2. 핵심 기술 (RAG Engine)**
* **Hybrid Search:** 의미 기반 검색(Dense)과 키워드 기반 검색(Sparse)을 결합하여 정확도 향상.
* **Semantic Routing:** LLM을 활용하여 사용자 질문의 의도를 파악하고 적절한 데이터 소스(공지, 학칙 등)로 라우팅.
* **Contextual Generation:** 검색된 관련 문서들을 문맥(Context)으로 제공하여 LLM이 정확한 답변을 생성하도록 유도.

**3. 주요 개선 사항**
* **세션 관리:** Redis를 도입하여 대화 맥락을 유지하면서도 메모리 효율성을 확보.
* **데이터 최신화:** 자동화된 크롤링 및 인덱싱 파이프라인을 통해 정보의 최신성 유지.

### 4. 기대효과

* **재학생 측면**
    * 신입생의 학교 적응 및 커리큘럼 로드맵 구성 지원.
    * 일관되고 신뢰성 있는 정보 제공을 통해 정보 혼란 방지 및 탐색 시간 절약.
* **학교 측면**
    * 디지털 역량 강화 및 대외적 위상 상승.
    * 단순 반복적인 문의 전화 감소로 인한 교내 행정 업무 효율성 향상.
